# Room App 第一版低保真原型

## 1. 页面范围

第一版需要这些页面：

- 登录页
- 注册页
- 忘记密码页
- 房间列表页
- 创建房间弹窗
- 房间详情页
- 个人资料页

第一版核心路径：

```txt
注册/登录 -> 房间列表 -> 创建或进入房间 -> 房间内开麦和文字聊天
```

├──────────────────────────────┤
│ 成员                          │
│ ┌──────┐ ┌──────┐ ┌──────┐   │
│ │头像   │ │空位   │ │空位   │   │
│ │Alex  │ │      │ │      │   │

│ └──────┘ └──────┘ └──────┘   │

│ 聊天                          │
│ ┌──────────────────────────┐ │
│ │ Mia：hello                │ │
│ │ Alex：hi                  │ │
│ │ 系统：Mia 进入房间         │ │
│ └──────────────────────────┘ │
│                              │
│ ┌──────────────────┐ ┌────┐ │
│ │ 输入消息，最多50字 │ │发送│ │
│ └──────────────────┘ └────┘ │
├──────────────────────────────┤
│ ┌──────────┐ ┌──────────┐   │

页面逻辑：

- 进入房间后建立 WebSocket
- WebSocket 断开等于离开房间
- 房主只展示身份
- 房主离开后，后端自动转让房主
- 最后一个成员离开后，房间和消息删除
- 文字消息通过 HTTP 发送，WebSocket 接收广播

## 9. 主要交互流

### 9.1 注册并进入房间

```txt
上传头像 -> 获取邮箱验证码 -> 注册 -> 房间列表 -> 创建房间 -> 房间详情
```

### 9.2 登录并进入房间

```txt
登录 -> 房间列表 -> 进入房间 -> 房间详情
```

### 9.3 房间内聊天

```txt
输入消息 -> HTTP 发送 -> 后端保存 -> WebSocket 广播 -> 房间成员收到消息
```

### 9.4 离开房间

```txt
点击退出 / WebSocket 断开 -> 后端删除成员记录 -> 必要时转让房主 -> 最后一人离开则删除房间
```
