# Room API 接口文档

## 1. 通用规则

接口前缀：

```txt
/api/v1
```

统一响应：

```json
{
  "code": 200,
  "message": "ok",
  "data": {}
}
```

鉴权请求头：

```txt
Authorization: Bearer jwt_token
```

免登录接口：

- `POST /api/v1/uploads/avatar`
- `POST /api/v1/auth/register-code`
- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/password-reset-code`
- `POST /api/v1/auth/reset-password`

## 2. 健康检查

```txt
GET /health
```

响应：

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "status": "up"
  }
}
```

## 3. 注册前上传头像

```txt
POST /api/v1/uploads/avatar
```

请求：

```txt
multipart/form-data
file: 头像文件
```

限制：

- jpg/jpeg/png/webp
- 最大 2MB
- COS 路径：`room/{uuid}.{ext}`

响应：

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "avatar_url": "https://your_bucket.cos.ap-guangzhou.myqcloud.com/room/uuid.png"
  }
}
```

## 4. 发送注册邮箱验证码

```txt
POST /api/v1/auth/register-code
```

请求：

```json
{
  "email": "alex@example.com"
}
```

响应：

```json
{
  "code": 200,
  "message": "ok",
  "data": null
}
```

## 5. 注册

```txt
POST /api/v1/auth/register
```

请求：

```json
{
  "username": "alex_001",
  "email": "alex@example.com",
  "email_code": "123456",
  "password": "123456",
  "nickname": "Alex",
  "avatar_url": "https://your_bucket.cos.ap-guangzhou.myqcloud.com/room/uuid.png"
}
```

响应：

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "token": "jwt_token",
    "user": {
      "id": 1,
      "username": "alex_001",
      "email": "alex@example.com",
      "nickname": "Alex",
      "avatar_url": "https://your_bucket.cos.ap-guangzhou.myqcloud.com/room/uuid.png"
    }
  }
}
```

## 6. 登录

```txt
POST /api/v1/auth/login
```

请求：

```json
{
  "username": "alex_001",
  "password": "123456"
}
```

## 7. 当前用户信息

```txt
GET /api/v1/users/me
```

需要 token。

## 8. 更新当前用户信息

```txt
PATCH /api/v1/users/me
```

请求：

```json
{
  "nickname": "Alex"
}
```

规则：

- 昵称 1-8 位
- 昵称不可与其他用户重复

## 9. 更新当前用户头像

```txt
POST /api/v1/users/me/avatar
```

请求：

```txt
multipart/form-data
file: 头像文件
```

需要 token。

## 12. 创建房间

```txt
POST /api/v1/rooms
```

请求：

```json
{
  "max_members": 8
}
```

规则：

- `max_members` 只能是 `2` 或 `8`
- 房间名称由后端生成：`用户昵称 + 的房间`
- 创建成功后，用户已经在房间内

## 13. 房间列表

```txt
GET /api/v1/rooms?page=1&page_size=20
```

规则：

- `page_size` 最大 50
- 按 `created_at` 倒序

响应：

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "list": [
      {
        "id": 1,
        "name": "Alex 的房间",
        "current_members": 1,
        "max_members": 8
      }
    ],
    "total": 1
  }
}
```

## 14. 房间详情

```txt
GET /api/v1/rooms/:room_id
```

返回房间信息和成员列表。

## 15. 进入房间

```txt
POST /api/v1/rooms/:room_id/join
```

规则：

- 房间存在
- 房间人数未满
- 用户当前不在其他房间
- 后端在事务中完成人数检查和成员创建

## 16. 离开房间

```txt
POST /api/v1/rooms/:room_id/leave
```

规则：

- 幂等
- 用户不在房间也返回成功
- 房主离开时自动转让房主
- 最后一个成员离开时删除房间和消息

## 17. 历史消息

```txt
GET /api/v1/rooms/:room_id/messages?limit=20&before_id=100
```

规则：

- 查询时按 `id` 倒序
- 返回给前端时按 `id` 正序

## 18. 发送文字消息

```txt
POST /api/v1/rooms/:room_id/messages
```

请求：

```json
{
  "content": "hello"
}
```

规则：

- 最多 50 字
- 后端 trim
- trim 后为空则拒绝
- HTTP 发送，WebSocket 广播

## 19. 更新麦克风状态

```txt
PATCH /api/v1/rooms/:room_id/mic
```

请求：

```json
{
  "mic_status": "on"
}
```

`mic_status`：

- `on`
- `off`

## 20. WebSocket

```txt
GET /ws/v1/rooms/:room_id?token=jwt_token
```

服务端事件：

- `member.joined`
- `member.left`
- `room.owner_changed`
- `message.created`
- `member.mic_updated`
