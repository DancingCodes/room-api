# Room 接口文档

## 通用规则

接口前缀：

```txt
/api/v1
```

服务端口固定为 `9999`。CORS 接受全部来源。

HTTP 响应统一格式：

```json
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```

业务 code：

- `200` 成功
- `401` 鉴权失败
- `500` 错误

HTTP 状态码统一返回 `200`，WebSocket 鉴权失败返回 HTTP `401`。

失败响应：

```json
{
  "code": 500,
  "message": "参数错误",
  "data": null
}
```

时间统一返回 ISO 8601 格式：

```txt
2026-06-16T16:00:00+08:00
```

鉴权请求头：

```txt
Authorization: Bearer jwt_token
```

参数规则：

- 字符长度按 Unicode 字符数计算
- 邮箱 trim 后转小写
- 账号正则为 `^[A-Za-z0-9_]{4,20}$`
- 昵称 trim 后 1-8 个字符
- 密码 6-20 位，密码不 trim
- 消息内容 trim 后 1-50 个字符

免登录接口：

- `POST /api/v1/uploads/avatar`
- `POST /api/v1/auth/register-code`
- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/password-reset-code`
- `POST /api/v1/auth/reset-password`

## 通用对象

用户：

```json
{
  "id": 1,
  "account": "alex_001",
  "email": "alex@example.com",
  "nickname": "Alex",
  "avatar_url": "https://example.com/room/uuid.png",
  "current_room_id": 1,
  "created_at": "2026-06-16T16:00:00+08:00",
  "updated_at": "2026-06-16T16:00:00+08:00"
}
```

`current_room_id` 表示当前用户所在房间 ID，没有所在房间时为 `null`。

`current_room_id` 不在 `users` 表存储，接口返回时从 `room_members` 查询。

房间：

```json
{
  "id": 1,
  "name": "Alex 的房间",
  "owner_id": 1,
  "current_members": 1,
  "max_members": 8,
  "created_at": "2026-06-16T16:00:00+08:00"
}
```

`current_members` 不在 `rooms` 表存储，接口返回时从 `room_members` 统计。
房主状态只以房间对象的 `owner_id` 为准；成员对象里的 `is_owner` 是接口返回时根据 `member.user_id == room.owner_id` 计算出来的字段，不在 `room_members` 表存储。

消息里的 `sender_nickname` 和 `sender_avatar_url` 使用发送者当前用户资料。

房间成员：

```json
{
  "user_id": 1,
  "nickname": "Alex",
  "avatar_url": "https://example.com/room/uuid.png",
  "is_owner": true,
  "mic_status": "off",
  "joined_at": "2026-06-16T16:00:00+08:00"
}
```

消息：

```json
{
  "id": 10,
  "room_id": 1,
  "sender_id": 1,
  "sender_nickname": "Alex",
  "sender_avatar_url": "https://example.com/room/uuid.png",
  "type": "text",
  "content": "hello",
  "created_at": "2026-06-16T16:00:00+08:00"
}
```

## 房间

### 创建房间

```txt
POST /api/v1/rooms
```

创建成功后用户已经在房间内，前端随后建立 WebSocket。

创建房间在事务中完成：检查用户是否已在房间、创建 `rooms`、创建 `room_members`。

请求：

```json
{
  "max_members": 8
}
```

响应：

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "room": {
      "id": 1,
      "name": "Alex 的房间",
      "owner_id": 1,
      "current_members": 1,
      "max_members": 8,
      "created_at": "2026-06-16T16:00:00+08:00"
    },
    "members": [
      {
        "user_id": 1,
        "nickname": "Alex",
        "avatar_url": "https://example.com/room/uuid.png",
        "is_owner": true,
        "mic_status": "off",
        "joined_at": "2026-06-16T16:00:00+08:00"
      }
    ]
  }
}
```

### 房间列表

```txt
GET /api/v1/rooms?page=1&page_size=20
```

分页：

- 默认 `page = 1`
- 默认 `page_size = 20`
- 最大 `page_size = 50`
- `page` 或 `page_size` 小于 1 时使用默认值
- `page_size` 大于 50 时按 50 处理
- 按 `rooms.created_at DESC` 排序

响应：

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "list": [
      {
        "id": 1,
        "name": "Alex 的房间",
        "owner_id": 1,
        "current_members": 1,
        "max_members": 8,
        "created_at": "2026-06-16T16:00:00+08:00"
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 20
  }
}
```

空列表响应：

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "list": [],
    "total": 0,
    "page": 1,
    "page_size": 20
  }
}
```

## 消息

### 历史消息

```txt
GET /api/v1/rooms/:room_id/messages?limit=20&before_id=100
```

要求当前用户已在该房间内。

分页：

- 默认 `limit = 20`
- 最大 `limit = 50`
- `limit` 小于 1 时使用默认值
- `limit` 大于 50 时按 50 处理
- 传入 `before_id` 时，返回 `id < before_id` 的消息
- 查询时按 `id` 倒序分页，返回时按 `id` 正序

响应：

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "list": [
      {
        "id": 10,
        "room_id": 1,
        "sender_id": 1,
        "sender_nickname": "Alex",
        "sender_avatar_url": "https://example.com/room/uuid.png",
        "type": "text",
        "content": "hello",
        "created_at": "2026-06-16T16:00:00+08:00"
      }
    ],
    "limit": 20
  }
}
```

空列表响应：

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "list": [],
    "limit": 20
  }
}
```

## WebSocket

```txt
GET /api/v1/ws/rooms/:room_id
Authorization: Bearer jwt_token
```

规则：

- 心跳使用 WebSocket ping/pong
- 服务端检测心跳超时后关闭连接，并执行离开逻辑
- `member.joined` 广播给加入成功时房间内已有 WebSocket 连接
- `message.created` 广播给房间内所有 WebSocket 连接，包括发送者
- `member.left`、`room.owner_changed`、`member.mic_updated` 广播给房间内剩余在线连接
- 主动离开和 WebSocket 断开共用同一套离开逻辑
- 离开逻辑幂等，只有实际删除成员记录的那一次广播 `member.left`
- 房主离开且房间仍有成员时，先广播 `member.left`，再广播 `room.owner_changed`
- `room.owner_changed.data.members` 返回房间内所有剩余成员

### member.joined

```json
{
  "type": "member.joined",
  "room_id": 1,
  "data": {
    "member": {
      "user_id": 2,
      "nickname": "Mia",
      "avatar_url": "https://example.com/room/mia.png",
      "is_owner": false,
      "mic_status": "off",
      "joined_at": "2026-06-16T16:01:00+08:00"
    },
    "current_members": 2
  }
}
```

### member.left

```json
{
  "type": "member.left",
  "room_id": 1,
  "data": {
    "user_id": 2,
    "current_members": 1
  }
}
```
