# Room 后端方案

下次接续开发时，先看 `PROJECT_MEMORY.md`。

## 目标

主流程：

```txt
注册/登录 -> 房间列表 -> 创建或进入房间 -> 房间详情
```

## 测试

普通测试：

```powershell
go test ./...
```

数据库集成测试默认跳过，避免误写线上数据库。需要准备专用测试库并执行初始化 SQL 后，再显式传入测试库 DSN：

```powershell
$env:ROOM_TEST_MYSQL_DSN="user:password@tcp(host:3306)/room_test?charset=utf8mb4&parseTime=True&loc=Local"
go test ./internal/repository -v
```

本地接口冒烟测试：

```powershell
.\scripts\api-smoke.ps1
```

WebSocket 冒烟测试：

```powershell
.\scripts\ws-smoke.ps1 -Username "你的用户名" -Password "你的密码"
```

第一版后端提供：

- 注册、登录、找回密码
- 当前用户资料

## 环境变量

```txt
JWT_SECRET=change_me
MYSQL_DSN=room:password@tcp(127.0.0.1:3306)/room?charset=utf8mb4&parseTime=True&loc=Asia%2FShanghai

TENCENT_SECRET_ID=your_tencent_cloud_secret_id
TENCENT_SECRET_KEY=your_tencent_cloud_secret_key
TENCENT_SES_REGION=ap-hongkong
TENCENT_SES_FROM=Room <moonc@ses.moonc.love>
TENCENT_SES_TEMPLATE_ID=186671

COS_BASE_URL=https://your_bucket.cos.ap-guangzhou.myqcloud.com
COS_PATH_PREFIX=room
COS_CDN_URL=https://your_cdn_domain
```

服务端口固定为 `9999`。CORS 接受全部来源。

数据库表结构使用 SQL migration 管理，初始化脚本为 `migrations/001_init_schema.sql`。

开发或部署前手动执行 `migrations/001_init_schema.sql`。服务启动时只连接 MySQL，不自动创建、不修改、不执行 migration。数据库连接失败时，服务启动失败并退出。

第一版不创建数据库外键，关联数据一致性由业务代码维护。

## 数据表

### users

```sql
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(20) NOT NULL,
  email VARCHAR(128) NOT NULL,
  nickname VARCHAR(8) NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  avatar_url VARCHAR(512) NOT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  UNIQUE KEY uk_users_username (username),
  UNIQUE KEY uk_users_email (email),
  UNIQUE KEY uk_users_nickname (nickname)
);
```

### email_verification_codes

```sql
CREATE TABLE email_verification_codes (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT UNSIGNED NULL,
  email VARCHAR(128) NOT NULL,
  purpose VARCHAR(32) NOT NULL,
  code VARCHAR(16) NOT NULL,
  used_at DATETIME NULL,
  expires_at DATETIME NOT NULL,
  created_at DATETIME NOT NULL,
  KEY idx_email_codes_email_purpose_created_at (email, purpose, created_at)
);
```

规则：

- `purpose` 为 `register` 或 `reset_password`
- 验证码 5 分钟有效
- 同一邮箱 60 秒内只能发送一次
- 同一邮箱 1 小时最多发送 5 次
- 同一邮箱同一用途重发后，旧验证码失效

### rooms

```sql
CREATE TABLE rooms (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(80) NOT NULL,
  owner_id BIGINT UNSIGNED NOT NULL,
  max_members TINYINT UNSIGNED NOT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  KEY idx_rooms_created_at (created_at)
);
```

`max_members` 只能是 `2` 或 `8`。

### room_members

```sql
CREATE TABLE room_members (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  room_id BIGINT UNSIGNED NOT NULL,
  user_id BIGINT UNSIGNED NOT NULL,
  mic_status VARCHAR(16) NOT NULL DEFAULT 'off',
  joined_at DATETIME NOT NULL,
  UNIQUE KEY uk_room_members_user (user_id),
  KEY idx_room_members_room_joined_at (room_id, joined_at)
);
```

`uk_room_members_user` 保证一个用户同一时间只能在一个房间。
Owner source of truth is `rooms.owner_id`; API member field `is_owner` is computed from `member.user_id == room.owner_id` and is not stored in `room_members`.

### messages

```sql
CREATE TABLE messages (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  room_id BIGINT UNSIGNED NOT NULL,
  sender_id BIGINT UNSIGNED NOT NULL,
  type VARCHAR(16) NOT NULL DEFAULT 'text',
  content TEXT NOT NULL,
  created_at DATETIME NOT NULL,
  KEY idx_messages_room_id_id (room_id, id)
);
```

## 业务规则

通用：

- 时间统一返回 ISO 8601 格式，例如 `2026-06-16T16:00:00+08:00`
- 服务时区使用 `Asia/Shanghai`
- HTTP 响应统一包含 `code`、`message`、`data`
- 字符长度按 Unicode 字符数计算
- 字符串字段按各业务规则 trim 后再校验和入库
- 邮箱 trim 后转小写
- 腾讯云 SES 和 COS 共用 `TENCENT_SECRET_ID`、`TENCENT_SECRET_KEY`
- 密钥只通过环境变量配置，不提交到仓库

用户：

- 用户名正则为 `^[A-Za-z0-9_]{4,20}$`
- 密码 6-20 位，使用 bcrypt 保存哈希，bcrypt 使用 Go 默认 cost，密码不 trim
- 邮箱唯一
- 昵称 trim 后 1-8 个字符且唯一
- 头像必填
- JWT 有效期 7 天
- JWT 签名算法使用 `HS256`
- JWT claims 至少包含 `user_id` 和 `exp`
- `current_room_id` 不在 `users` 表存储，接口返回时从 `room_members` 查询

验证码：

- 验证码为 6 位数字
- 发送新验证码时，将同一邮箱同一用途的旧验证码 `used_at` 更新为当前时间
- 校验验证码时，只接受未使用、未过期的最新一条同用途验证码
- 发送重置密码验证码时，如果邮箱不存在，也返回成功
- 60 秒和 1 小时发送频率按 `email + purpose + created_at` 统计，不受 `used_at` 影响
- 验证码邮件使用腾讯云 SES API 发送
- 发信地域为中国香港，`TENCENT_SES_REGION = ap-hongkong`
- 发信域名为 `ses.moonc.love`
- 发信地址为 `moonc@ses.moonc.love`
- 发件人格式为 `Room <moonc@ses.moonc.love>`
- 发件人显示名由 `TENCENT_SES_FROM` 控制
- 邮件模板名称为 `Room账号验证码`
- 邮件模板 ID 为 `186671`
- 邮件模板变量为 `code`
- 模板类型为纯文本
- 模板正文为 `您的 Room 验证码为：{{code}}。验证码 5 分钟内有效，请勿向他人泄露。如非本人操作，请忽略本邮件。`
- 发送时调用腾讯云 SES `SendEmail`，`TemplateData` 传 `{"code":"123456"}`
- 邮件发送封装为独立服务，业务层只调用发送验证码方法
- 邮件发送超时时间为 10 秒
- 验证码邮件发送失败不自动重试
- 邮件发送失败时，接口返回错误，不写入新的验证码记录

头像：

- 支持 jpg、jpeg、png、webp
- 最大 2MB
- 文件类型使用 MIME sniff 校验
- COS 路径为 `{COS_PATH_PREFIX}/{uuid}.{ext}`
- 扩展名统一小写
- 上传使用 `COS_BASE_URL`
- 返回 URL 优先使用 `COS_CDN_URL + "/" + object_key`，未配置 `COS_CDN_URL` 时使用 `COS_BASE_URL + "/" + object_key`
- 第一版公开读

房间：

- 房间名称为 `用户昵称 + 的房间`
- 房间名称创建后固定
- 创建房间后用户自动成为房主
- 创建房间需要在事务中完成：检查用户是否已在房间、创建 `rooms`、创建 `room_members`
- 创建房间和进入房间的 HTTP 接口成功后，用户已经在房间内，前端随后建立 WebSocket
- 一个用户同一时间只能在一个房间
- 房间详情接口要求当前用户已在该房间内
- 进入房间需要在事务中锁定 `rooms` 行，检查人数并创建成员
- 房主离开后，最早加入的成员成为新房主
- 最后一个成员离开后，删除房间和消息
- 房间列表默认 `page = 1`、`page_size = 20`，最大 `page_size = 50`
- 房间列表按 `rooms.created_at DESC` 排序
- 房间成员列表按 `room_members.joined_at ASC` 排序
- 房主状态只以 `rooms.owner_id` 为准；接口成员字段 `is_owner` 根据 `member.user_id == room.owner_id` 计算，不在 `room_members` 存储
- 分页参数小于 1 时使用默认值，大于最大值时按最大值处理
- `current_members` 不在 `rooms` 表存储，接口返回时从 `room_members` 统计

消息：

- 第一版只支持文本
- 内容 trim 后不能为空
- 最多 50 字
- HTTP 发送，WebSocket 广播
- 历史消息查询时倒序分页，返回时按 id 正序
- 历史消息默认 `limit = 20`，最大 `limit = 50`
- 历史消息接口要求当前用户已在该房间内
- 消息接口返回的 `sender_nickname` 和 `sender_avatar_url` 使用发送者当前用户资料

WebSocket：

- 地址为 `/api/v1/ws/rooms/:room_id`
- 使用 `Authorization: Bearer jwt_token` 请求头鉴权
- 连接前校验 token、房间和成员身份
- 断开等于离开房间
- 心跳间隔 30 秒，超时 60 秒
- 心跳使用 WebSocket ping/pong
- 服务端检测心跳超时后关闭连接，并执行离开逻辑
- `member.joined` 广播给加入成功时房间内已有 WebSocket 连接
- `message.created` 广播给房间内所有 WebSocket 连接，包括发送者
- `member.left`、`room.owner_changed`、`member.mic_updated` 广播给房间内剩余在线连接
- 主动离开和 WebSocket 断开共用同一套离开逻辑
- 离开逻辑幂等，只有实际删除成员记录的那一次广播 `member.left`
- 房主离开且房间仍有成员时，先广播 `member.left`，再广播 `room.owner_changed`
- `room.owner_changed.data.members` 返回房间内所有剩余成员

## WebSocket 事件

- `member.joined`
- `member.left`
- `room.owner_changed`
- `message.created`
- `member.mic_updated`

统一格式：

```json
{
  "type": "event.name",
  "room_id": 1,
  "data": {}
}
```
