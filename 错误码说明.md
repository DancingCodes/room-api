# Room 错误码说明

## 1. 业务 code

第一版只使用三类业务 code。

```txt
200 成功
401 鉴权失败
500 错误
```

## 2. 成功

```json
{
  "code": 200,
  "message": "ok",
  "data": {}
}
```

## 3. 鉴权失败

```json
{
  "code": 401,
  "message": "unauthorized",
  "data": null
}
```

触发场景：

- token 为空
- token 无效
- token 过期
- WebSocket token 无效

## 4. 错误

```json
{
  "code": 500,
  "message": "error message",
  "data": null
}
```

触发场景：

- 参数错误
- 用户名已存在
- 邮箱已存在
- 昵称已存在
- 验证码错误或过期
- 登录失败
- 房间不存在
- 房间已满
- 用户已在其他房间
- 邮箱发送失败
- COS 上传失败
- 数据库错误

## 5. HTTP 状态码

第一版 HTTP 状态码统一返回 `200`，通过业务 `code` 判断结果。

WebSocket 鉴权失败时，不升级连接，直接返回 HTTP `401`。
