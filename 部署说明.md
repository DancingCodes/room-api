# Room 部署说明

## 1. 第一版部署目标

第一版后端部署到云服务器，连接云 MySQL、腾讯云 COS 和 QQ 邮箱 SMTP。

Android App 通过后端 API 地址访问服务。

## 2. 部署前准备

需要准备：

- 云服务器
- 云 MySQL 数据库
- database：`room`
- 腾讯云 COS bucket
- QQ 邮箱 SMTP 授权码
- 后端环境变量
- Android API Base URL

## 3. 后端部署步骤

1. 上传后端代码
2. 配置 `.env`
3. 编译 Go 服务
4. 启动服务
5. 检查 `/health`
6. 检查 AutoMigrate 是否成功
7. 检查日志

## 4. 环境变量

参考：

[环境变量说明.md](./环境变量说明.md)

## 5. 数据库

云数据库需要提前创建：

```txt
room
```

服务启动后自动执行 GORM AutoMigrate。

如果数据库连接失败或 AutoMigrate 失败，服务直接退出。

## 6. COS

COS 要求：

- bucket 可访问
- 头像公开读
- 后端拥有上传权限
- 文件路径：`room/{uuid}.{ext}`

## 7. 邮箱

QQ 邮箱要求：

- 开启 SMTP
- 使用授权码
- 不使用 QQ 密码

## 8. Android 配置

Android 需要配置后端 API 地址：

```txt
https://api.example.com
```

WebSocket 地址：

```txt
wss://api.example.com/ws/v1/rooms/{room_id}?token=jwt_token
```

开发环境可以使用 HTTP/WS，正式环境建议使用 HTTPS/WSS。
